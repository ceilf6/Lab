# browser

这个目录是浏览器相关知识与文档的组织入口，内容围绕 **Web API、JavaScript 学习、前端知识导航** 以及 **浏览器存储（storage / cookie）** 等主题展开。它更像一个学习路线与知识边界的总览，而不是单纯的 API 逐项说明手册。适合 **前端开发者、JavaScript 初学者、Web 开发学习者**，以及希望系统理解浏览器运行环境、接口调用方式、前端能力边界与常见存储机制的人。即使在 AI 工具快速发展的背景下，这些基础知识仍然有助于建立稳定的前端认知框架。

## Detailed Description

### [1] `Web-API/`

这是一个围绕 **Web API、JavaScript 学习、前端知识导航** 的入口型目录，主要作用不是提供完整的 API 教程或逐项参考文档，而是帮助读者快速理解 `../JS/Web-API` 相关内容的学习方向与知识边界。目录内容强调：即使在 **AI 快速发展** 的背景下，Web API 的实际使用场景可能发生变化，但理解浏览器能力、接口调用方式、前端运行环境与知识体系依然很有价值。

它尤其适合：
- **前端开发者**：用于梳理浏览器提供的能力与常用接口；
- **JavaScript 初学者**：帮助建立从语言到平台 API 的过渡认知；
- **Web 开发学习者**：理解页面运行时、事件、网络、存储、DOM 等相关概念；
- **知识导航型读者**：希望先看全局，再深入某个具体 API 或主题。

从结构上看，这一目录更偏向“导航页”或“学习入口”，核心价值在于帮助你判断：哪些内容属于浏览器原生能力，哪些是前端日常开发中高频使用的接口，以及这些知识在当前技术环境下的学习意义。

### [2] `store/`

这是仓库中与 **浏览器存储** 相关的文档组织区域，重点围绕 **cookie 处理** 及其延伸内容展开。它更像一个专题聚合目录，用于整理浏览器端存储机制、会话数据、持久化方案以及与 cookie 相关的资源或说明。

从目录定位来看，`store/` 不是独立项目，而是整个 `browser` 主题下的重要组成部分，属于前端浏览器知识体系中的“状态与数据保存”模块。它适合关注以下内容的读者：
- **浏览器 cookie** 的原理、用途与使用场景；
- **前端存储机制** 的分类与差异理解；
- **Web 开发学习者** 想要掌握用户状态管理、会话维持与数据持久化；
- **前端工程实践者**，用于理解登录态、偏好配置、跨页面数据保存等常见问题。

如果说 `Web-API/` 更偏向浏览器能力全景，那么 `store/` 就更聚焦于“浏览器如何保存数据、如何处理状态”的专题方向。它通常会与登录、会话、跨页面通信、缓存和前端数据管理等知识点形成关联，是浏览器基础知识中很实用的一环。
