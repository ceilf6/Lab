# README

This directory contains the main documentation hub for **libuv**, a cross-platform asynchronous I/O library used by Node.js and other runtime systems. It includes the project overview, feature highlights, licensing, versioning and ABI stability notes, documentation build guidance, release verification steps, platform support details, and practical build/test instructions for UNIX-like systems, macOS, Windows, and package managers such as Homebrew, vcpkg, and Conan. The content is especially useful for developers, contributors, maintainers, and porters who need to understand how to build libuv, run its tests and benchmarks, or work with platform-specific constraints.

### Build_Instructions.md Build_Instructions.md

This file is a practical **build and testing guide** for libuv. It explains how to compile, install, and validate the project across multiple environments, including **UNIX-like systems**, **macOS**, and **Windows**. It covers both major build systems used by the project: **autotools** and **CMake**. For Windows, it outlines prerequisites and notes that CMake is the supported route.

The guide also includes **cross-compilation** guidance and package-manager-based installation paths through **Homebrew**, **vcpkg**, and **Conan**, making it useful for developers working in varied toolchains. Beyond building, it provides instructions for running the test suite, invoking individual tests, and using debugging tools such as **gdb** and **valgrind**. It also mentions running benchmarks and warns that some tests are timing-sensitive, which is important when diagnosing failures or flaky behavior.

Overall, this is the main hands-on reference for anyone who needs to build libuv locally, verify changes, or debug issues in a development environment.

### Supported_Platfor_2more_12fc8c90.md Supported_Platfor_2more_12fc8c90.md

This document summarizes **platform support** and related build considerations for libuv. It is focused on compatibility across operating systems and toolchains, with specific notes for **AIX** and **z/OS**. The document also references the project’s broader supported platforms guidance and includes compiler-related advice, especially the recommendation to use **`-fno-strict-aliasing`** for safer interaction with libuv’s API patterns.

The platform-specific sections explain that **AIX** requires **IBM XL C/C++ 12.1+** and **bos.ahafs**, while **z/OS** needs **ZOSLIB** and CMake-based configuration. It also notes a cleanup consideration on z/OS involving **System V resources** and the use of `ipcrm`. A short **Patches** section points contributors toward the project’s contribution guidelines, indicating that this file is not just a compatibility reference but also a pointer for maintainers and porters working on platform adaptations.

This document is most relevant for developers, systems engineers, and contributors who are building libuv on less common platforms or validating compiler and OS support assumptions.
