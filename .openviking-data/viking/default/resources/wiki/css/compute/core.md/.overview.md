## Detailed Description

### core.md `core.md`

这份笔记是一个面向 CSS 样式计算机制的教程，围绕浏览器在渲染前如何确定元素最终样式展开。它不是简单罗列概念，而是按处理流程逐层推进，帮助读者建立“浏览器如何做决定”的整体认知。

首先，文档从 **“确定声明值”** 开始，说明浏览器会先收集某个元素在各来源中的所有样式声明，再通过层叠规则筛选出候选结果。这里强调了 CSS 冲突处理的三大关键因素：**重要性（important）**、**特殊性（specificity）** 和 **源代码次序（source order）**。这部分适合用来理解为什么某些样式明明写了却不生效，以及为什么权重更高的规则会覆盖其他声明。

接着，笔记进入 **“层叠冲突”** 的分析，说明当多个声明作用于同一属性时，浏览器如何比较规则并决定最终采用哪一个声明值。它帮助读者从“写法优先”转向“规则优先”，理解 CSS 不是简单的后写覆盖前写，而是一个带有优先级判定的系统。

然后，文档讲解 **“使用继承”**，重点说明某些属性会从父元素继承计算结果，而不是每个元素都独立指定。这里特别提到继承不是万能的：并非所有属性都默认继承，只有符合继承规则的属性才会沿着 DOM 树向下传播。文中还举例说明了 `a` 元素颜色继承的限制，提醒读者某些标签会有浏览器默认样式或特殊行为，不能只凭直觉判断。

最后，笔记介绍 **“使用默认值”**，解释当属性没有声明值、也不能继承时，浏览器会回退到该属性的默认值或初始值。这里重点涉及 `initial` 和 `inherit` 两个特殊取值：`initial` 用于强制回到属性的初始状态，`inherit` 则显式要求继承父元素对应属性值。通过这些内容，读者可以更清楚地理解 **computed value** 的形成过程，以及属性在渲染前是如何一步步被确定的。

整体来看，`core.md` 适合作为 CSS 样式系统的基础进阶笔记，帮助前端学习者把“样式为什么这样算”这件事从经验层面提升到机制层面。
