# navigation-l1

This directory is a reference-style L1 navigation guide for the OpenViking workspace, focused on helping agents and technical readers quickly decide where to inspect next without loading large source trees. It covers workspace map navigation, repository indexing, frontend and browser/runtime areas, AI-related notes, sandbox entrypoints, and retrieval defaults. The content is especially useful for repository explorers, AI agents, and engineers working with React, Vue, Chromium, Node.js, V8, LangChain, and other large source mirrors. It emphasizes high-signal markdown notes, learning folders, runnable sandboxes, and safe ingestion priorities for efficient source-reading and code-impact analysis.

## Detailed Description

### navigation-l1.md `navigation-l1.md`

This file is a high-level L1 navigation guide for the OpenViking workspace. Its purpose is not to teach the codebase in depth, but to help agents and readers choose the next best place to inspect. It organizes the workspace into major areas such as **Root**, **AI**, **React**, **Vue**, **Browser And Runtime**, **Frontend Knowledge Notes**, **Sandbox And App Entrypoints**, and **Retrieval Defaults**. Each section points toward high-signal markdown notes, focused learning folders, runnable sandboxes, and large source mirrors with explicit ingestion guidance.

A major feature of this file is its prioritization rules. It tells readers to begin with `README.md`, `docs/WORKSPACE_MAP.md`, and this navigation guide itself, then move to concise reference files like `core.md`, `README.md`, `.flows`, and `.effect-pictures` before opening large source mirrors. It also frames large mirrors such as **React**, **Vue**, **Chromium**, **Node.js**, **V8**, and **LangChain** as opt-in and topic-scoped rather than default starting points. For code-impact and symbol-level investigation, it directs users toward **GitNexus**.

The file is designed for AI agents, technical readers, and repository explorers who already have basic familiarity with frontend systems, browser/runtime concepts, and source-reading workflows. In practical terms, it functions as a repository index, workspace map, navigation guide, and retrieval policy document all in one. Keywords include navigation guide, workspace map, repository index, source mirror ingestion, React, Vue, browser, Node.js, AI infrastructure, and retrieval defaults.
