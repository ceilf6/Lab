# benchmark-questions

这是一个面向 **检索评测（retrieval benchmark）** 的参考目录，核心目标是在导入 **OpenViking 索引** 后，测试文档检索是否能够从 **L0/L1 导航信息** 准确收敛到高信号文件，而不是被大体量源码镜像干扰。内容围绕 10 个 benchmark questions 展开，覆盖 **React beginWork、CSS BFC/视觉格式化模型、FrontAgent RAG 与上下文管理、OpenClaw/agent skill、浏览器存储、Vue2/Vue3 响应式、Node/CommonJS、网络安全与跨域、sandbox 运行入口，以及 GitNexus/OpenViking 职责划分** 等主题。它适合做 **知识库检索、仓库导航设计、RAG 评测、前端/浏览器/Node.js/安全基础能力验证** 的开发者或研究者使用。

## Detailed Description

### benchmark-questions.md `benchmark-questions.md`

该文档是整个目录的核心文件，定位为一份 **检索评测参考指南**。它并不是单纯的题目列表，而是围绕“如何检索到正确高信号文档”构建的一套评测框架：当索引中同时存在导航文档、概览页、以及大体量源码镜像时，系统应优先命中 **L0/L1 导航信息**，而不是直接落到原始代码细节。

文档包含 **10 个 benchmark questions**，覆盖多个常见技术域：  
- **React**：如 `beginWork` 的执行逻辑与相关渲染路径  
- **CSS**：BFC、视觉格式化模型等布局机制  
- **RAG / Agent**：FrontAgent 的上下文管理、OpenClaw 的 agent skill  
- **浏览器存储**：localStorage、sessionStorage、IndexedDB 等概念性检索  
- **Vue 响应式**：Vue2 / Vue3 的实现差异与依赖追踪  
- **Node.js**：CommonJS 模块系统与加载流程  
- **网络安全**：跨域、同源策略、权限与边界  
- **Sandbox 入口**：运行入口与执行链路  
- **仓库职责划分**：GitNexus 与 OpenViking 的关系和分工

每道题都强调一个关键原则：**理想答案应该先命中导航、概览或说明性文件**，如果只返回源码，则需要补充对应的 **L1 导航笔记** 才算通过。换言之，这份文档更像是一份“检索质量验收规范”，可用于评估索引结构、召回策略、排序策略，以及是否具备“从目录到高价值文档”的收敛能力。

适合人群包括：  
- 做 **RAG / 搜索 / 知识库检索** 的工程师  
- 设计仓库信息架构、目录导航、文档索引的人  
- 需要评测模型或搜索系统是否“会找文档”的研究者  
- 具备一定 **前端、浏览器、Node.js、网络安全** 基础的开发者
