# asset-index

## Detailed Description

### asset-index.md `asset-index.md`

这是整个目录的核心索引文档，定位为“参考型入口页”。它的作用不是展开讲解某个技术原理，而是把 OpenViking 中零散分布的可视化资产、主题目录和检索入口集中整理，帮助读者在最短时间内找到想看的图、目录或专题。

内容结构上，文档大致覆盖以下几类主题：

- **React Flow 图与相关图片**：聚合 React 渲染与更新流程、Fiber、Scheduler、生命周期、patch flag 等相关图示资源，强调先通过图理解主流程，再进入源码或专题。
- **Vue 资产**：提供 Vue 相关图示或入口，便于与 React 主题并列检索，适合对比框架机制。
- **CSS 与 HTML 主题入口**：包括 BFC、布局等关键概念，偏向从目录和图示中快速定位对应知识块。
- **Browser 与 NodeJS 运行时入口**：覆盖 cookie、localStorage、CommonJS、libuv 等关键词，便于从运行时和宿主环境角度查找材料。
- **AI/Agent 相关应用入口**：涵盖 RAG、MCP 等方向，说明该索引不仅服务前端和运行时，也面向新一代 AI 应用检索。
- **Retrieval Notes**：文末提供检索建议，强调阅读优先级与路径，例如 React beginWork 流程优先查看 `.flows/react_beginWork_flow.drawio`，CSS BFC 优先看 `CSS/BFC/`。

这个文件的价值在于“导航效率”。如果你已经知道自己要找的是哪个主题，它会告诉你应该先看哪里；如果你还不确定，它也能通过关键词和目录结构帮你建立检索路径。整体上，它适合源码阅读者、前端/浏览器/Node.js 学习者，以及需要在大量资产中快速定位信息的 AI 辅助分析场景。
