## Detailed Description

### frontagent-l1.md `frontagent-l1.md`

This file is a stable reference and navigation document for the FrontAgent repository. It is designed to help readers orient themselves across the project’s major topics—RAG, memory, context, and architecture—without requiring immediate deep code inspection.

The document is organized around several practical entry paths:

- **Product and Architecture Docs**: points to the main README, architecture notes, Chinese-language docs, and a memory design analysis to provide a broader conceptual foundation.
- **RAG Implementation Entrypoints**: identifies where the retrieval flow begins, especially in the `mcp-memory` `rag.ts` layer.
- **Memory and Context Entrypoints**: maps the parts of the codebase responsible for memory handling and context construction, including core memory and context modules.
- **Retrieval Notes**: explains how to approach tracing depending on whether the user is asking about RAG behavior, memory design, or MCP tools.

It also references the key implementation files involved in remote repository RAG, MCP client integration, CLI configuration, cache commands, and agent-level context injection. A notable emphasis is placed on **GitNexus** for symbol-level tracing, while **OpenViking** is described as a navigation/retrieval layer rather than the primary implementation target.

Overall, this file functions as a routing guide for the repository: it tells developers where to start, how to narrow their search, and which code paths or docs are most relevant to their question. It is particularly valuable for maintainers, RAG/MCP investigators, and contributors who need fast, reliable orientation across the FrontAgent architecture.
