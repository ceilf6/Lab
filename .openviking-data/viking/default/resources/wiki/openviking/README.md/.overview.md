## Detailed Description

### README.md README.md

这份文档是一份 **OpenViking 渐进式探索接入指南**，核心目标是说明如何将 OpenViking 作为仓库的**旁路上下文层**，用于知识库探索、目录级摘要、语义检索和分层上下文构建，同时与 **GitNexus** 分工协作，分别承担知识探索与代码调用关系/影响分析。

文档首先阐明接入目标与适用场景：OpenViking 更适合做 **目录级摘要、语义搜索、分层上下文组织**，可以避免把大型源码镜像直接灌入造成噪声过高的问题。随后给出本地试点部署步骤，包括如何生成本地配置、启动服务，以及如何完成首批资源摄入。文中强调 **manifest 摄入** 是资源管理的重要机制，并说明了如何通过 `add_resource` 等方式把资源纳入上下文层。

在能力要求上，文档明确指出 **embedding 是必需项**，用于构建基础检索能力；**VLM** 可作为多模态增强；**rerank** 则用于提升搜索结果质量。为了便于本地闭环验证，还提到可以使用 **hash embedding shim** 进行测试，但生产环境应使用真实模型接口。对 API 兼容性也有要求，提及 **OpenAI-compatible API** 形态，便于快速对接现有客户端和服务。

在接入方式上，README 详细介绍了 **MCP** 的使用方法，以及 OpenViking 内置的工具调用范式，包括 `search/find`、`glob`、`remember` 等，用于检索、模式匹配与上下文记忆。文档还给出了使用规则与验收方法，帮助用户确认从资源摄入到检索、再到上下文组织的完整链路是否可用。整体而言，这是一份面向 **开发者、AI 工具集成者、知识库/文档维护者** 的接入型指南，强调从本地试点到逐步推广的工程实践路径，并附带参考链接便于进一步深入理解。
