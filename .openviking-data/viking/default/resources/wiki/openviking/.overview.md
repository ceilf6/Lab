# openviking

OpenViking 是一个面向 **仓库探索、知识索引、分层导航与检索评测** 的参考型工作区，整体更像一套“旁路上下文层”而不是单纯的源码仓库。它围绕 **L0/L1/L2 导航、asset 索引、ingestion manifest、MCP 接入、embedding、VLM、多模态检索、rerank** 等主题组织内容，帮助开发者、AI Agent 集成者和技术读者快速定位高信号信息，避免被大体量源码镜像淹没。该目录尤其适合需要做 **知识库摄入、语义检索、workspace 导航、检索评测** 的读者，以及希望将 OpenViking 与 GitNexus 协同使用的人。

## Detailed Description

### [1] `navigation-l1.md/`
这是 OpenViking 工作区的 **L1 导航指南**，属于参考型导航层。它的核心目标不是解释源码实现，而是帮助人或 Agent 在不展开大体量目录树的情况下，快速判断下一步该看哪里。内容重点包括 **workspace map navigation、仓库概览、分层检索路径、优先级判断** 等，适合在探索初期作为“路标”使用。对于需要从广泛主题中收敛目标文件的场景，它是很重要的入口。

### [2] `asset-index.md/`
这是 OpenViking 的 **参考型资产索引目录**，用于把分散的二进制图片、图示文件和稀疏目录中的可视化资产统一编目。它更偏向 **导航与检索地图**，而不是原理教程；重点是目录组织、资产映射和优先查看顺序。其内容提到 React Flow、Vue、CSS/HTML、Browser、NodeJS、AI/Agent 等相关主题，说明它既服务于前端/浏览器知识定位，也服务于多模态或图示化资料检索。适合想快速找到可视化线索的人。

### [3] `frontagent-l1.md/`
这是一个面向 **FrontAgent** 的 L1 级导航目录，但当前摘要显示为 **“directory abstract is not ready”**。可以将其视为 FrontAgent 相关内容的预留入口或待补充导航点。结合仓库整体主题，它很可能用于承载与 **RAG、上下文管理、Agent 工作流、前端知识接入** 相关的指引。对于寻找 FrontAgent 主题的人来说，这里更像一个待完善的分层索引节点。

### [4] `README.md/`
这是 OpenViking 仓库的核心说明目录，介绍如何把 OpenViking 作为仓库的 **旁路上下文层** 来做知识库探索、分层检索和语义理解，并与 **GitNexus** 配合处理代码级调用关系和影响分析。摘要明确提到 **L0/L1/L2 分层上下文、manifest 摄入、MCP 接入、embedding、VLM、多模态检索、rerank、本地试点部署、服务启动与验收** 等内容。适合开发者、AI 客户端集成者，以及负责文档与检索系统落地的技术人员。

### [5] `benchmark-questions.md/`
这是一个面向 **检索评测（retrieval benchmark）** 的参考目录，主要用于验证 OpenViking 索引是否能把查询从 L0/L1 导航信息准确收敛到高信号文件。它的重点不是功能实现，而是评估检索质量，避免被大体量源码镜像干扰。摘要提到的主题覆盖 **React beginWork、CSS BFC/视觉格式化模型、FrontAgent RAG 与上下文管理、OpenClaw/agent skill、浏览器存储、Vue2/Vue3** 等，说明它兼顾前端基础、Agent 语义与仓库检索能力测试。

### [6] `ingest-manifest.tsv/`
这是用于 **仓库索引、内容摄取与检索映射** 的配置参考目录，描述 OpenViking / GitNexus 如何把 Markdown、目录和特定资源纳入知识库。内容覆盖 **kind 规则、target 目标映射、exclude 排除策略**，并列出 README、AGENTS、docs、AI、CSS、HTML、Browser、NodeJS、React、Vue、TS、sandboxs 等高价值文档与主题目录的收录范围。它适合需要配置入库范围、控制索引噪声、建立稳定检索入口的工程人员。
