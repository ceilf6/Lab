# README

这是一个基于 Vite 的实验性 playground 仓库入门说明，面向前端开发者、学习者以及需要搭建本地实验环境的工程师。它围绕 **playground、sandbox、monorepo、HMR、ES modules、快速开始** 等关键词，介绍如何在统一的项目结构中组织多个开发实验，并通过路由快速访问不同 sandbox。仓库强调模块化实验管理、自动索引、独立的 HTML/CSS/JS 结构以及高频迭代能力，适合已有基本 npm、Vite 和现代 JavaScript 经验的人阅读与使用。

## Detailed Description

### README.md README.md

该文件是整个仓库的主入口文档，承担“项目介绍 + 使用指南 + 功能概览”的作用。它首先说明这是一个 **基于 Vite 的实验性 playground**，用于学习、练习和管理多个模块的开发实验，核心组织方式是将每个实验放在 `sandboxs/<name>/` 下，并通过对应路由访问。文档明确给出默认运行地址 `localhost:8030`，并提到仓库中包含 `sandboxs/demo` 示例，帮助新用户快速理解目录结构和访问方式。

内容结构上，README 主要分为四块：

- **Modules**：介绍仓库关注的三个方向——**Sandboxes、Agent、Monorepo**，说明项目不仅是单一 demo，而是一个可扩展的实验管理框架。
- **Quick Start**：提供基础启动流程，包括安装依赖、启动开发服务器以及创建新实验的步骤，适合第一次接触该仓库的开发者。
- **Available Commands**：列出常用命令，如 `dev`、`build`、`preview`，对应开发、构建和预览场景。
- **Features**：总结项目特性，包括 **HMR 热更新、自动扫描 sandboxs、独立 HTML/CSS/JS、ES modules 支持、首页自动索引** 等，体现了这个 playground 的快速迭代和模块化实验能力。

从使用价值上看，这份 README 不只是“怎么跑起来”，更是在说明这个仓库如何作为一个 **Vite playground / sandbox 管理系统** 来组织多个实验。它特别适合需要在同一代码库中快速试验前端方案、对比多个模块实现，或者希望建立一个可持续扩展的实验型 monorepo 的开发者。
