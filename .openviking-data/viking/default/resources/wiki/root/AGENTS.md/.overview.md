## Detailed Description

### AGENTS.md AGENTS.md

`AGENTS.md` is the central **project guidance and operational reference document** for the repository. It defines the expected workflow for working with GitNexus-powered code intelligence, including how to **explore the codebase**, **analyze symbol relationships**, **trace execution flows**, **perform impact analysis**, and **commit safely**. The document serves as a procedural handbook rather than application code, and it assumes familiarity with Git, source symbols, runtime behavior, and MCP-style tooling.

A major theme of the file is **safety-first repository interaction**. It establishes “Always Do” and “Never Do” rules, including the requirement that **impact analysis must be completed before editing any function, class, or method**. It also requires **change detection before committing**, ensuring that modifications are reviewed and understood before being finalized. If any **high or critical risk** is identified, that warning must be surfaced to the user clearly.

The document also explains the repository’s **index status**, noting that it is indexed as **“Lab”** and advising users to verify **index freshness** before conducting analysis. This is important for ensuring that symbol lookups and context retrieval reflect the current state of the codebase.

Another key area is the distinction between **GitNexus** and **OpenViking**. GitNexus is presented as the tool for **symbol-aware analysis**, contextual code intelligence, and precise repository navigation. OpenViking is positioned as a companion resource for **exploratory knowledge-base navigation** and documentation ingestion. This separation helps users choose the right system depending on whether they need deep code understanding or broader informational discovery.

The file also references several **CLI skill files** that support common engineering tasks, including:
- **architecture exploration**
- **impact analysis**
- **debugging**
- **refactoring**
- **tool usage**

Overall, `AGENTS.md` functions as the **authoritative operating guide** for developers and agents working in the repository. It is especially valuable for those performing analysis, making changes, or using GitNexus/OpenViking workflows in a controlled and reproducible way.
