# README

这是一个用于 **Node.js Proxy Server** 的项目说明文档，重点介绍如何搭建一个支持 **HTTP/HTTPS 转发** 与 **WebSocket 代理** 的本地代理服务器，并完整记录请求与响应日志。内容覆盖 **环境变量配置、快速开始、Claude Code 使用方式、日志格式说明、示例与 License**，适合需要调试网络请求、排查接口问题、查看完整请求链路日志的开发者和运维人员阅读。文档强调代理不修改转发内容，支持保存请求/响应的 headers、body、状态码与耗时，并具备日志文件自动轮转能力，适用于本地开发、代理调试和自动化工具接入场景。

## Detailed Description

### README.md

该文件是整个仓库的核心指南，面向使用者完整说明一个 **Proxy Server** 的设计、配置与操作方式。文档首先概述项目能力：作为一个基于 **Node.js** 的代理服务器，它可以转发 **HTTP/HTTPS** 请求以及 **WebSocket** 连接，并且在转发过程中保持内容不被修改，适合需要原样透传流量的调试和观测场景。

随后，文档详细列出关键环境变量，包括 **TARGET_URL**、**PORT**、**LOG_DIR**、**LOG_LEVEL** 等，帮助用户指定目标服务、监听端口、日志目录及日志级别。对于本地开发和部署，这部分是最关键的配置入口，适合需要快速调整代理行为的人。

在 **快速开始** 部分，README 提供安装、启动和开发模式相关命令，帮助用户尽快把代理服务跑起来。无论是第一次接触该项目，还是需要在新环境中重新部署，都可以通过这部分直接完成基础搭建。

文档还专门加入了 **Claude Code 使用** 说明，展示如何在该工具相关的工作流中通过代理绕行或临时关闭代理，以便更灵活地管理网络请求路径。这一部分说明了项目不仅面向通用代理场景，也考虑了特定开发工具的集成需求。

在 **日志说明** 中，README 详细说明日志会记录的内容，包括 **请求方法、Headers、Body、响应状态、耗时** 等信息；其中 Body 会同时保存 **decoded body** 与 **raw body 的 base64** 表示，便于在编码或二进制场景下完整回溯原始请求内容。同时，日志文件支持自动管理，最多保留 **20 个**，体现了对长期运行场景的维护友好性。

最后的 **示例** 与 **License** 部分进一步补充了实际使用方式与授权信息。整体来看，这份文档非常适合 **前端/后端开发者、测试人员、DevOps、运维工程师** 使用，尤其是那些需要搭建本地代理、调试接口、审计请求日志、分析 WebSocket 流量的用户。
