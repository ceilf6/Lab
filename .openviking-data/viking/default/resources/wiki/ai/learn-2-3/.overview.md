## Detailed Description

### [1] README.md/

这是一份面向入门学习的索引型说明文档，围绕“输入 -> Model -> 输出”的视角帮助读者理解第二、第三层相关概念，以及数据处理和网络观察的基本流程。内容重点集中在两条主线：一是抓包工具与方法，推荐使用 Proxyman 作为实践入口；二是代理服务器相关概念，并提到 `_proxy-server` 作为辅助工具或项目线索。整体更像是学习路线提示、实践入口与概念导航，而不是完整教程。适合正在学习网络抓包、代理服务器、Model 处理过程以及基础层级概念的初学者，尤其适合希望通过工具实操建立直观理解的读者。

### [2] GLOBAL-AGENTS.md/

`GLOBAL-AGENTS.md` 是一个面向全局代理行为约束的规范目录，核心内容围绕“遇到不确定情况时不要自行决定，而要先向用户提问并确认”这一决策原则展开。它强调澄清、确认、审慎执行和避免擅自推断，体现出一种交互式、低误判的工作流思路。这个目录适用于 AI 代理、自动化助手、文档维护规范或协作开发场景，尤其适合需要严格控制行为边界、减少歧义决策的系统或团队。虽然内容精简，但关键词明确：GLOBAL-AGENTS、用户提问、确认、澄清、不确定性、决策规范。

### [3] REP-AGENTS.md/

`REP-AGENTS.md` 更像是一份仓库级协作手册或工程参考规范，重点覆盖项目约定、代码开发流程、技术栈与工程结构、Git 提交规范等内容。它的作用是帮助开发者、维护者和新加入项目的协作者快速对齐仓库的组织方式与工作标准，从而在开发、提交、协作和维护时保持一致性。这个目录适合具备基础 Git 和软件工程协作知识的读者，尤其适合需要了解仓库规则、开发流程和提交约定的人。它和整个 `learn-2-3` 仓库的关系更偏向“协作标准层”，为其他学习或实现内容提供统一行为框架。

### [4] skill/

`skill/` 是一个高层知识或文档空间，指向仓库中与“技能”相关的内容组织。根据现有结构判断，这里目前没有独立源文件，更像是一个用于承载学习资料、能力模块、方法论或实践笔记的目录容器。它通常不会被理解为独立项目，而是整个仓库知识体系的一部分，承接与技能积累、学习主题或能力分层相关的内容。对于想了解仓库如何按主题组织知识、或者希望扩展某类技能文档的读者，这里是一个潜在入口。

### [5] proxy-server/

`proxy-server/` 是仓库中与代理服务器主题直接相关的目录，可能用于承载文档、配置、实现代码或实验性内容。结合当前目录整体的学习导向，它更像是与抓包、网络观察、代理机制、请求转发等主题相配套的实践空间，而不是完全独立的外部项目。这个目录与 `README.md/` 中提到的代理工具与网络学习路径相呼应，说明仓库可能希望通过代理服务的理解和实践，帮助读者掌握输入输出链路、网络请求流向和中间层观察方法。适合对网络代理、调试、中间件行为感兴趣的学习者和开发者。
