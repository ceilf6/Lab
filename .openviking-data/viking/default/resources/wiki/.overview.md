# wiki

这个 `wiki` 目录是一个面向前端、Web、浏览器、Node.js、Vue、React、TypeScript、CSS、AI 与知识检索等主题的综合知识库入口，更像一套“技术地图 + 学习导航 + 源码阅读索引”。它把不同技术栈、实验区、文档区、图片资料和工作流目录组织在一起，适合前端开发者、源码阅读者、技术写作者、学习者，以及需要快速定位某个技术主题资料的人。整体内容偏向文档驱动、知识沉淀和目录化导航，而不是单一业务应用。

## Detailed Description

### [1] `docs/`
围绕前端工程实践、Vite 实验场、源码阅读与知识地图的文档集合，覆盖 playground、sandbox、monorepo、HMR、ES modules、组件库构建、JavaScript 基础与前端研究导航等内容，也包含对 Ant Design Mobile、JavaScript 对象模型等具体技术点的说明。

### [2] `ts/`
一个 TypeScript 方向的工作区，结构上看像 `Vite + React` 项目脚手架或示例模板，适合做前端开发、起步工程、实验项目或教学示例。

### [3] `sandboxs-runner/`
轻量级仓库快照，根部没有源码文件，主要包含子工作区 `wordMCP/`，整体更像文档优先或早期阶段项目，可能与某个 MCP/word 相关实验有关。

### [4] `vue2/`
Vue 2 相关的知识与参考工作区，包含源码学习、文档、概念说明等支持材料，适合做 Vue 2 原理学习和组件机制研究。

### [5] `vue3/`
Vue 3 主题工作区，聚焦框架内部机制、源码理解与生态材料，适合研究 Vue 3 的响应式、组件体系与工程实践。

### [6] `react/`
React 相关知识仓库，重点可能包括 sandbox 示例、React 原理、调度与 Fiber、Hooks 等内容，偏向学习与源码分析而非生产应用。

### [7] `html/`
HTML 文档与内容型目录，按主题组织，整体像 wiki 或生成站点结构，适合查找前端基础、网页结构和内容参考资料。

### [8] `css/`
CSS 高层知识入口，包含若干聚焦子目录，适合学习样式系统、布局、视觉表现与 CSS 相关工程知识。

### [9] `browser/`
浏览器知识导航，围绕 Web API、JavaScript 学习、浏览器存储、cookie、storage 等主题展开，适合理解浏览器运行环境和前端能力边界。

### [10] `nodejs/`
Node.js 相关目录，核心围绕 `libuv`、事件循环、网络、文件系统和系统级异步 I/O 能力，是理解 Node.js 底层机制的重要入口。

### [11] `ai/`
以“输入 → Model → 输出”为主线的学习型知识目录，涉及网络基础、数据处理流程、代理服务器理解和协作规范，适合 AI/网络基础入门。

### [12] `effect-pictures/`
前端原理图与技术图解资料目录，覆盖 React、Vue、Vite、JavaScript、CSS、模块打包、跨域、消息通道、Transformer 等主题，适合做分享、复习与快速理解。

### [13] `openviking/`
面向仓库探索、知识索引、分层导航与检索评测的参考型工作区，重点围绕 L0/L1/L2 导航、asset 索引、ingestion manifest、MCP 接入、embedding、VLM 与多模态检索等能力。

### [14] `root/`
仓库级知识地图与工作流指南，覆盖 Vue2/Vue3、React、Scheduler、Hooks、事件循环、Webpack/Vite、Node.js、设计模式、架构设计，以及 RAG/Agent 与安全协作规范。

### [15] `obsidion/`
轻量级文档型仓库，当前主要内容组织在 `README.md/` 子目录下，整体偏向笔记、知识库或说明文档聚合。

### [16] `flows/`
一个空的流程占位目录，可能用于工作流定义、自动化资产或 Flow 配置，目前尚未包含实际内容。
